Development Graph

The development strategy used in MoodSwing is similar to the one prescribed in Game Architecture and Design by Rollings and Morris. The model introduced by this book is a Tier-based Model, wherein each tier represents a certain level of completion — the game is playable at the end of each tier.

Below is a graph representing the development progress. The development process is divided into three phases: the design phase, the development phase, and the testing phase. Nodes represent a certain area of development. Green nodes represent finished areas. Yellow nodes represent areas that are not yet finished but are currently being worked on. Red nodes represent unfinished areas that are currently not being worked on. Click on nodes to view details relating to the corresponding area.
